This document includes information for .NET TX developers about how to customize $(TD_APP_NAME).

Customizing $(TD_APP_NAME) for Drawings.NET Classic

Preparing Drawings.NET Classic files for $(TD_APP_NAME)

  1. Download and unzip the ODA SDK Xtension (TX) SDK from www.opendesign.com.
  2. In the unzipped "exe" folder, navigate to your configuration folder, and copy TD_Mgd_version.dll and TD_MgdBrep_version.dll to the folder where $(TD_APP_NAME) is installed.
  3. In the unzipped "Platforms.NET_Classic" folder, navigate to your configuration folder, and run the .sln file.
  4. Build the solution. You can build all of the solutions or only the examples you need. However, you must build the NETLOADCommand example, which is required for loading any Drawings.NET Classic module.
    Note: ODA SDK TX provides two Drawings.NET Classic examples: Create3dSolids and ManagedTestCommand.
  5. Copy the built .dll and .tx modules, including NETLOADCommand.tx, to the folder where $(TD_APP_NAME) is installed.

Loading Drawings.NET Classic modules in $(TD_APP_NAME)

  1. Start $(TD_APP_NAME).
  2. Do one of the following:
    • Choose Tools > Load Application.
    • Type APPLOAD in the command line.
  3. Navigate to the NETLOADCommand.tx file and click Load, then repeat for all Drawings.NET Classic modules (built .dll and .tx files).
    Note: You can also click Startup Suite to load the modules each time $(TD_APP_NAME) starts.

    Modules that are loaded successfully display in the Loaded Applications list.
  4. Click Close.
  5. Type NETLOAD in the command line.
  6. In the Assembly Path dialog box that displays, choose your managed .tx or .dll file and click Open to load it.

Now you can run commands from the loaded modules. For example:

  • Create3dSolids, which is implemented in the Create3dSolids module.
  • AddSomeText, which is implemented in the ManagedTestCommand module.

You can create your own modules or modify an existing module for your purposes. To start with an existing module, use the solution for your configuration, for example, "Platforms.NET_Classic\vc10dll\ODA_vc10dll.sln". Open the solution with Microsoft Visual Studio. See also the example projects in "NET\ManagedCommands."
